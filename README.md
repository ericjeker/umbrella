# Zig Umbrella

Umbrella Project for exploring Zig

## What this is

A minimal SDL3 hello-world application written in Zig, built for both Linux
and Windows from a single `zig build`. SDL3 is compiled from source as a Zig
package (`castholm/SDL`). The same `src/main.zig` compiles unchanged for both 
targets via `@cImport` of SDL3's C headers.

## Requirements

- **Zig 0.16.0** installed locally and on `PATH`.
- On Linux, the X11/Wayland client libraries must be present at runtime
  (SDL3 dlopens them). They ship with most distros and are present on WSL.

## Build

```bash
zig build                       # build both targets
                                # → zig-out/bin/hello_sdl3      (Linux)
                                # → zig-out/bin/hello_sdl3.exe  (Windows)

zig build -Doptimize=ReleaseFast  # release build (much smaller binaries)
zig build --summary all           # build with a per-target summary tree
```

The Windows `.exe` is produced via cross-compilation (target
`x86_64-windows-gnu`) using the mingw-w64 toolchain bundled with Zig.

## Run

```bash
zig build run                   # build + run the native Linux binary
```

Under WSL, the Wayland socket isn't where SDL3 expects it. Use:

```bash
XDG_RUNTIME_DIR=/mnt/wslg/runtime-dir WAYLAND_DISPLAY=wayland-0 zig build run
```

Or persist those exports in `~/.bashrc`. To run without any display
(smoke test using SDL3's dummy video driver):

```bash
SDL_VIDEODRIVER=dummy ./zig-out/bin/hello_sdl3
```

To run the Windows binary from WSL, invoke it directly (WSL hands it to
Windows as a native process):

```bash
./zig-out/bin/hello_sdl3.exe
```

## Format and lint

Zig ships a non-configurable formatter and an AST-level checker. There is
no separate linter — what would be warnings in other languages are compile
errors in Zig.

```bash
zig fmt src/ build.zig build.zig.zon            # format in place
zig fmt --check src/ build.zig build.zig.zon    # CI mode: exit non-zero if unformatted
zig fmt --check --ast-check src/ build.zig      # format check + AST-level error check
zig ast-check src/main.zig                      # standalone AST check on one file
```

## Test

Unit tests live next to the code as `*_test.zig` files (e.g.
`src/entities/triangle_test.zig`) and are wired into `build.zig` as a
`test` step. Run them with:

```bash
zig build test                  # compile + run all unit tests
zig build test --summary all    # same, plus a per-test pass/fail summary
```

Tests import the module under test by relative path and exercise its
public surface. SDL is linked into the test binary because modules under
test import SDL for drawing, even when the test itself only touches pure
math. See `build.zig`'s `test` step and `addSdlModule` helper for the
wiring.

## Project layout

```
<root>
├── build.zig        # build script: Linux + Windows exes, `test` step, links SDL3
├── build.zig.zon    # manifest: pins the SDL3 package (URL + content hash)
├── src/
│   ├── main.zig     # SDL3 entrypoint: setup, event loop, dispatch to entities
│   ├── sdl.zig      # single shared @cImport of SDL3 (imported as the "sdl" module)
│   └── entities/
│       ├── triangle.zig      # Triangle struct + rotate/draw
│       └── triangle_test.zig # unit tests for Triangle
├── AGENTS.md        # full context for AI agents working on this repo
└── .gitignore       # ignores .zig-cache/, zig-out/, zig-pkg/
```

`.zig-cache/`, `zig-out/`, and `zig-pkg/` are generated by `zig build` and
should not be committed. They are restored from `build.zig.zon` on demand.

## Dependencies

SDL3 is declared in `build.zig.zon` under `.dependencies.sdl`, pinned to a
specific git commit + content hash. `zig build` fetches it automatically
into `zig-pkg/` on first build. To update or remove it, see
[AGENTS.md](AGENTS.md#updating--removing-sdl3).
